proof

the function must return a value of y

because it does not know anything about y : it does not how to create an y, it can not create a new y by its own

the only possibility left is to use its second input, which is of type x -> y that and only that can produce an y

it has to apply that x -> y function

the only possibility to apply it is feeding to it a value of type x

our x * (x -> y) -> y function has access to one and only one x, which is its first input
