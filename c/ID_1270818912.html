function application

function application is a very common kind of expression in programming, especially in functional programming languages hence many language designers think it deserves the most simple syntax :

f x

this is the invocation of function f with argument x

this expression may be read in english as both of these

function f is applied to argument x

argumnet x is applied to function f

"parameter" or "argument" ?

a parameter belongs to the definition of a function

an argument belongs to an application of a function

an argument is an actual value of a parameter

{ "parameter" "argument" } -> "input"

it may be advisable to change the naming as follows :

"parameter" -> "formal input"

"argument" -> "actual input"

the reason is : "parameter" is also often used in a different and important sense

see parametricity

implicit arguments

functions may declare some of their inputs implicit typical such ones are type parameters at call site the actual argument of these can be guessed by the compiler, for example from the explicitly given arguments implicit arguments may still be given explicitely with a special syntax : enclosed by curly brackets, like "f {a1 a2}" example : cosine {Float} 0.0
