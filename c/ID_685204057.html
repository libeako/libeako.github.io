is a method to branch on the structure of a data value and give new names to parts of it the structure of the value is matched to the structure of a "pattern" (sub-values in the value) are matched to (the symbols in the pattern) by this giving names to those sub-values, making us able to refer them some people say pattern matching "destruct"s the value this phrase may be slightly misguiding for novices of course : the value as an object in memory does not get destroyed or modified by pattern matching

example of simple pattern matching

the data type, a value of which is to be pattern matched

Pair = Man * Woman

the value to be pattern matched

a_pair = (adam, eva)

the pattern matching itself

(xy, xx) = a_pair

here "(xy, xx)" is the pattern, in which xy, xx are the symbols that to be the names of the sub-values of the value, which in this case are the components of the pair

now

xy denotes adam xx denotes eva

it is equivalent to introducing these names one-by-one

xy = get 1 a_pair xx = get 2 a_pair

already you should be able to see one advantage of pattern matching : it is often shorter, more readable, less error-prone

composite pattern matching

branching

the following example implements the well-known if-then-else function

if_then_else : Bool -> T -> T -> T if_then_else c a b = case c true -> a false -> b

the 'case' keyword introduces the "case expression" the value after the 'case' keyword is matched to patterns the evaluation continues on the branch the pattern of which manages to match

the compiler should check in compile time that at least one of the patterns will match in runtime, with other wording : that the pattern matching is exhaustive

warning : the haskell compiler ghc does not check this by default, you have to turn it on with an argument to the compiler

pattern matching can reach sub-values at any depth

size : List e -> Integer size l = case l nil -> 0 head :: tail -> 1 + (size tail)

the following example implements a size function for Lists which takes a List of Maybes and counts only the existing elements

effective_size : List (Maybe a) -> Int effective_size l = case l nil -> 0 (just _) :: tail -> 1 + (size tail) none :: tail -> size tail

in this example pattern matching reaches not only the elements of a List value, but also inside the Maybe structure under it

simultanious

matching multiple values simultaniously is possible by building a tuple of those values and matching the tuple the following example hopefully demonstrates why sometimes this is useful

unsure_sum : Maybe Int -> Maybe Int -> Maybe Int unsure_sum a b => case (a, b) | (just x, just y) => just (x + y) | _ => none

some programming languages [for example haskell] provide a syntax sugar : allow the pattern matching to be moved from the top of the right side of "=" to the left side of it]

unsure_sum : Maybe Int -> Maybe Int -> Maybe Int unsure_sum | (just x) (just y) = just (x + y) | _ _ = none
