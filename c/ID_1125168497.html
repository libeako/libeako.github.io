type annotation

(x is a value of type t) is usually written as "x : t"

haskell, purescript use double ":"

idris, agda use single ":"

as type systems get more capable, the use of type annotation is getting more important, it deserves the simple syntax

mathematical type theory uses single ":"

symbol definition

(s is a symbol of type t with value v) is written as follows

in mathematical text

s : t := v

many practical programming languages use simply "=" instead of ":="

some programming languages make it possible to separate the type annotation from the value assignment into separate lines

s : t s = v

let-expression

an expression letting the coder declare a new symbol with scope limited to the expression

example

let square x = x * x in (square a) + (square b)

"let", "in" are keywords

a "simple type" is one given by the underlying programming language semantics

a "constructed type" is one that is constructed of simpler types

type construction can be

structural

such type is defined solely by the structure of the construction

nominal

such type is defined by the structure of the construction and a name for the new type 2 of them with the same structure but with different names are considered to not equal
