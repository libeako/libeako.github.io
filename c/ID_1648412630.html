SimCo is to be a simple universal configuration language

the name "SimCo" is from phrase "simple configuration"

the current state of configuration languages

the well known configuration languages are dhall, json, yaml, toml

if you know about other noteworthy one then tell me about it

dhall has a professional and principled design it is simple too, but only relatively to its expressive power and safety but absolutely it is complex, heavy it is a real programming language though total, but still a programming language i can imagine complex configuration tasks where that power is good but many [perhaps the majority of] configuration tasks require only a few simple unrelated properties, expressed as strings for those tasks involving an almost full programming language seems to me too much with my respect towards the professional design work behind dhall : i think there is place for something order of magnitude dumber

json is not for humans its syntax is heavy, and does not support comments

yaml is complex

toml comes close to what i intend but i think an even simpler language is possible

the description and design rational of simco

built-in types

there is no principled set of types to have the privilege of being built-in

if boolean is built-in, then why not 3-valued logical values [true, false, unknown] too?

if integer numbers are built-in, then why not rational, then floating point, then complex, then ... numbers too?

if date is built-in, then why not IP addresses, postal addresses, ... too?

there is no principle that can draw the boundary hence simco does not have any built-in types, everything is just string, so far as simco is concerned

basic syntax

a basic configuration file consists of a list of properties, their keys and values, separated by " = "

name = banana color = yellow taste = sweet

keys may contain only regular letters, decimal digits, '-'

keys may not be empty

for backward and other safety reasons : the values may not contain leading or ending whitespace

neither the keys nor the value are in apostrophes this is to keep the syntax clean this suffices, because in the majority of cases leading and ending whitespace are not present in the values if they are then the user can apply whatever apostrophe mechanism it likes

for example

apostrophe = " \" "

here '" \" "' is the value of the node so far as simco is concerned and the user can parse this value further to extract ' " '

the value ends at the new-line character hence multi-line values are not supported directly

multi-line values are not normal in configurations and when one is needed then it is probably healthier anyway to have it in a separate text file and only its file path in the configuration file

tree structure

the simple properties can be organized into categories

boot loader = grub2 command = print "hello master"; system-package-manager --update network machine-name = main-pc

the names of the categories are keys and have the same syntax as keys of simple properties

indentation with tabs encode the parent-child relationships between the nodes

categories can be nested in whatever tree structure

system boot loader = grub2 command = print "hello master"; system-package-manager --update network machine-name = main-pc applications = geany firefox tor-browser users - name = john real-world-name = John Doe - name = jane real-word-name = Jane Doe

any multiplicity of the keys is allowed

any branch can be disabled entirely by making its root "disabled" with the "/ " prefix

system boot loader = grub2 command = print "hello master"; system-package-manager --update network machine-name = main-pc applications = geany firefox tor-browser / users - name = john real-world-name = John Doe - name = jane real-word-name = Jane Doe

here the "users" branch is disabled

comments

the basic piece of a comment is a comment node, introduced by "// "

key = value // this is a comment

comments can be nested

key = value // this is a comment // and this one is too

comments must be in the tree structure this syntactic framework of the comments enables us to programmatically modify configuration files without messing up the comments

further ideas

i feel that these ideas are not worth the extra complexity :

replace indentation with composite keys

example

write

system boot loader = grub2 command = print "hello master" system network machine-name = main-pc

instead of

system boot loader = grub2 command = print "hello master" network machine-name = main-pc

perhaps this may be useful to make deep trees more human readable

the recommended file name extension for simco is ".simco.text"
