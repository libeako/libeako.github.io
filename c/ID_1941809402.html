type classes are type interfaces that use static method dispatch

type classes do not increase the expressive power of the language, do not enable additional abstractions they only make the coding work more convenient, less verbose

type classes are very popular and it looks that in the future they will be part of most well designed programming languages

haskell

type classes were introduced by the haskell programming language and haskell always remained to be the testbed for new research in this topic lately popular industrial languages also started to adopt it : rust's "traits" and c++'s "concept"s are type classes under different names

type class vs OO type interface

the roles of type classes

a type class can be seen as all of these :

interface for types, similarly to as in java, c#

as such : they provide us the ability to implicit argument passing

partial function from the set of types

as special case : mapping types to their default values

via this the language can provide the ability of implicit arguments to functions

a principled method overloading technique

a syntax of type classes

laws

laws are intended mathematical properties [usually equalities] of interface implementations that though can not be stated and checked by the used static typ-system cheaply enough : they are still to be obeyed by implementations they are written only in comment or documentation and must be obeyed by discipline of the software coder

example :

the laws of interface Equatable should be the usual mathematical properties of equality : equivalence, substitutivity

not all interfaces have to have laws
