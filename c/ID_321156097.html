the set of data types contructable with product and sum type construction is named "algebraic data types", commonly shortened to "adt"

adts may have a type parameter

type Maybe A = | none | just A

or multiple type parameters

type Result A B = | error A | success B

off topic : 'Result' is a generalization of 'Maybe', storing the reason of non-existence

'Either' is a conceptual generalization of 'Result'

they are equivalent mathematically but 'Either' is a name that do not restrict to result of computation

type Either L R = | left L | right R

'Either'' can represent 'Result' a slight ergonomy problem : in this role : it is not straightforward which case to mean success and which to mean error to help in this : one may think about the 'right' case as the right [as "okay" or "successful"] case with this ergonomy problem solved : the name "Either" should be preferred over "Result", because of its conceptual generality which is often used in practice

'Either' is commonly considered to be the sum type dual of the 2-ary product type [the pair] and hence also named with (+)

the (names of the cases of a sum type) [like "none", "just", "left", "right"] also serve as the names of corresponding constructor functions of the sum type

algebraic data types can be recursive

for example the linked list

type List A = | nil | cons A (List A)

the "cons" constructor is often replaced with an operator

type List A = | nil | A :: List A

to enable easy construction of list values

list : List Number list = 1 :: 2 :: 3 :: nil
