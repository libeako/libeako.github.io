sum type is a mathematical dual of product type

this suggests it is important it is in convenience

introduction via examples

it is a generalization of what is known as "union" or "enum" type in c, java, c#

type Color = | green | blue | red

just as an interesting example : the Bool type can be defined with it :-)

type Bool = | false | true

the number of cases may = 1

type Singleton = | single

exactly 1 value exists with type Singleton it is 'single'

the number of cases may = 0

type Void =

exactly 0 value exists with type Void

such type is sometimes named "Absurd"

the generalization relative to the "enum" type is that it can store value in the cases

type Shape = | point Vector

here the case 'point' stores a value of type 'Vector'

or multiple values

type Shape = | point Vector | circle Vector Float

here the case 'circle' sotres a value of type 'Vector' and a value of type 'Float'

storing multiple values in a single case is mathematically equivalent to storing a single product

type Shape = | point Vector | circle (Vector * Float)

we can give names to the stored values, just like in record types

type Shape = | point Vector | circle (center : Vector) (radius : Float) | line (normal : Vector) (distance_from_origo : Float)

the size [as the number of values] of a sum type is the sum of the sizes of its member [case] types hence the name "sum type"

sum type is similar to subtyping

their comparison reveals a duality between them, which helps to decide on when to use which method

sum type is closed in cases, open in methods

a sum type can not be extended with a new case the sum type's definition enumerates all the cases

new methods processing values of the sum type can be freely created

subtyping is open in cases, closed in methods

new subtypes of a supertype [like a java type interface] can be freely added

the supertype's definition fixes the set of methods that can be uniformly applied to a value of the supertype
