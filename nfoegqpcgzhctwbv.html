"magma" means just a binary function (t -> t -> t)

it is rather known as a type class:

class Magma t where

(<>) : t -> t -> t

some famous potential properties of magmas

associative: (x <> y) <> z = x <> (y <> z)

commutative: x <> y = y <> x

idempotent: x <> x = x

absorb

a value x is "idempotent" under a function f iff (f x x = x)

a function is idempotent iff all values are idempotent under it
